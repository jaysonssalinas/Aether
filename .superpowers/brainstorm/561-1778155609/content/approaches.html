3 Approaches to the Aether Architecture

All achieve 1 domain + local CRM. They differ in how much restructuring is involved.

A

Minimal Merge Recommended

Delete apps/digital and apps/celebrations — they're redundant since apps/aether already has /digital/* and /celebrations/* routes fully built. Deploy only apps/aether to Netlify. Set up apps/admin on Ubuntu + Tailscale. Done in 1 session.

Pros

  • Fastest path to live
  • No content migration risk
  • Already works today

Cons

  • Assumes apps/aether has all content needed
  • Need to verify nothing is missed
B

Content Audit Merge

Audit apps/digital and apps/celebrations for any unique pages or components not yet in apps/aether. Port those in, then delete the subdomain apps. More thorough — ensures nothing is left behind.

Pros

  • No risk of losing content
  • Clean final result

Cons

  • Takes longer
  • May find duplicate work
C

Full Restructure (Follow the MD Doc)

Rebuild to match the MD document exactly: websites/, admin-dashboard/, api/, database/. Replaces the current Next.js monorepo with the planned structure from scratch.

Pros

  • Matches your original vision doc
  • Clean slate

Cons

  • Throws away working code
  • Weeks of rework
  • High risk, low reward
